Accessibility Audit Case Study

This case study documents an accessibility audit I performed on my own contact form page. The goal was to identify issues that could affect users with disabilities and implement improvements based on WCAG guidelines.

Audit Scope

  • Keyboard navigation
  • Screen reader compatibility
  • Form labels and semantics
  • Color contrast
  • Focus visibility

Issues Found

  1. Missing or unclear form labels
    Some inputs did not have properly associated <label> elements.
  2. Insufficient focus styling
    Interactive elements did not have a clear visible focus indicator.
  3. Button contrast could be improved
    The button color contrast was borderline for WCAG AA.
  4. No success message after submission
    Users did not receive confirmation that their message was sent.

Fixes Implemented

  • Added explicit <label> elements with matching for attributes
  • Improved focus styles for keyboard users
  • Adjusted button colors to meet WCAG AA contrast
  • Added a simple, accessible success message

What I Learned

This audit helped me better understand how small issues can create barriers for users with disabilities. I gained experience identifying accessibility problems, applying WCAG guidelines, and improving the usability of a real component.
